vue +nodejs 博客后台管理系统

linjhon 2017/5/20 VueexpressMongoDB

一个 Vue + nodejs + MongoDB 后台管理系统。到现在大概 5 天的时间，先写个总结。

Vue 负责前端页面的展示，路由功能交给 Vue-router。

Express 不写路由，只负责写 api 接口， 使用 mongoose 与 MongoDB 数据库进行数据交互，实现前后端解耦。

在 UI 框架的选型上还是不太满意，写过之后才发现 Iview 框架有不少的缺陷，比如 menu 菜单中缺少路由跳转功能，需要通过点击事件返回的name值，通过编程式导航的方式来进行路由跳转router.push()，再来就是文档写得并不是很好，虽然 ui 效果很好。

mongodb 是非关系型数据库，通常我们可以在里面直接存储 json，对象，图片，甚至是函数。这对于前端来说，会更加方便。在操作 mongodb 数据库时，推荐使用 mongoose 的。我们可以通过定义 Schema 来配置一个集合里的数据类型(同 SQL 的表)，再把这个 Schema 发布成 Model，然后我们就可以通过这个 Model 来进行增删改查。

# 技术

前端

vue2 + vuex + vue-router + iview + axios/vue-axios + ES6

后端

nodejs + express +mongoose

工具

webpack

数据库

MongoDB

# 项目预览

本地需要安装 mongodb 数据库

# 客户端 client

# 打开目录
 cd client

# 安装依赖包
 npm i

# 启动本地客户端服务器
npm run dev

# 打包文件
npm run build

# 服务端 server

# 打开目录
 cd server

# 安装依赖包
 npm i

# 开启服务器
 npm run start

# 功能

  • 用户、文章、评论实现增删改查；
  • 基于 JWT 的登陆验证、状态留存；
  • 使用 muler 实现上传图片功能；
  •  多分类管理（曾删改）；

# 更新计划

  • 优化分类， 与各模块进行交互；
  • 首页模块；
  • 数据统计模块；
